5. 学習ロードマップ（AI 前提バージョン）

だいたいこんなフェーズで進めるのが現実的だと思う：

Phase 1: Java の「方言」を覚える（〜 基本文法 + OOP）
	•	Python との対応を意識しながら、AI にこういうことをさせる：
	•	「Python のこのクラスを Java に翻訳して」
	•	「この Java コードを、Pythoner 向けに解説して」
	•	ここでやること：
	•	基本文法（変数、if、for、配列、List など）
	•	クラス・継承・インターフェース
	•	public / private / protected / package-private
	•	パッケージと import

Phase 2: Gold 範囲のコア（設計・ジェネリクス・コレクション）
	•	抽象クラス vs インターフェース
	•	内部クラス・匿名クラス
	•	ジェネリクス（境界つき型パラメータ、ワイルドカード）
	•	コレクション（ArrayList / HashMap など）と Comparator

ここは デザインパターン経験が活きるゾーンだから、
AI に「Strategy パターンを Java で実装して」「ジェネリクスを使ったリポジトリクラス書いて」とか頼んで、
自分で写経 → 改造 → 質問 というループを回す。

Phase 3: ラムダ・Stream・例外・I/O・並行処理
	•	ラムダ式 / メソッド参照
	•	Stream API（中間操作・終端操作）
	•	例外階層、checked / unchecked、try-with-resources
	•	I/O / NIO（Files, Path, BufferedReader など）
	•	Thread / ExecutorService / Future / ロック系

ここからは コードを動かしながらじゃないと絶対に身につかないので、
	•	自分で小さな問題を作る（「テキストファイルを読み込んで、行数を数える」とか）
	•	まず自力で Java コードを書く
	•	動かしてみて、エラーも含めて AI に見せてレビューさせる

というスタイルがベスト。

Phase 4: 試験問題形式のトレーニング
	•	公式トピック一覧に沿って、トピックごとに AI に 4択問題を量産させる
	•	解く → 答え合わせ → なぜ他の選択肢がダメかを AI に説明させる
	•	間違えたものは mistakes-log.md に記録して、定期的に復習
